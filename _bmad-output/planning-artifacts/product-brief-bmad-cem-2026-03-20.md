## Executive Summary

CEM-приложение для сбора обратной связи с визуальным граф-конструктором опросов на базе React Flow. Продукт создаётся как собственный инструмент с полным контролем данных на своём сервере. Ключевое отличие — наглядный визуальный конструктор с динамическим бранчингом, где граф IS порядок прохождения опроса. Первый пользователь — автор продукта; при успешной валидации — выход на рынок стартапов с фокусом на лёгкость и скорость.

---

## Core Vision

### Problem Statement

Существующие платформы для сбора фидбэка (Typeform, SurveyMonkey, Google Forms) не дают полного контроля над данными — всё хранится на чужих серверах. Условная логика опросов скрыта за списками правил и неинтуитивна. Нет инструмента, который бы совмещал визуальную наглядность построения опроса с гибким динамическим бранчингом.

### Problem Impact

Без собственного инструмента сбор фидбэка не ведётся вообще. Данные респондентов остаются на сторонних платформах без контроля владельца. Сложная условная логика опросов создаётся вслепую — без визуальной верификации путей прохождения.

### Why Existing Solutions Fall Short

- **Нет контроля данных** — данные хранятся на серверах третьих сторон
- **Скрытая логика бранчинга** — условия задаются через правила и списки, а не визуально
- **Нет валидации путей** — невозможно увидеть тупиковые ветки или циклы до запуска
- **Нет аналитики на графе** — отсутствует визуализация путей прохождения (heatmap)

### Proposed Solution

Multi-tenant SaaS-приложение с визуальным конструктором опросов на React Flow. Каждый вопрос — нода графа с типизированными выходами. Граф определяет порядок и логику прохождения. Встроенная валидация обнаруживает тупики и циклы. Аналитика отображается прямо на графе в виде heatmap путей. Данные хранятся на сервере владельца (PostgreSQL). Стек: Next.js + NestJS + TypeORM, monorepo на Turborepo.

### Key Differentiators

- **Визуальный граф-конструктор** — React Flow вместо скрытых правил, граф = порядок опроса
- **Типизированные выходы нод** — NPS автоматически разветвляется на detractor/passive/promoter
- **Валидация флоу** — обнаружение тупиков и циклов перед активацией опроса
- **Heatmap на графе** — визуализация путей прохождения респондентов прямо на конструкторе
- **Полный контроль данных** — всё на своём сервере, никаких третьих сторон

## Target Users

### Primary Users

**Владислав — владелец SaaS-продукта**

- **Контекст:** Владелец и оператор SaaS-бизнеса с базой клиентов, которую нужно регулярно опрашивать на предмет удовлетворённости
- **Роль в системе:** Organization Owner — единственный пользователь на старте, создаёт опросы, рассылает ссылки, анализирует результаты
- **Мотивация:** Понимать, насколько клиенты довольны работой с его основным продуктом. Выявлять проблемные зоны через NPS, распределение ответов и satisfaction matrix
- **Текущая ситуация:** Фидбэк не собирается вообще — нет подходящего инструмента с контролем данных и визуальным конструктором
- **Ключевой сценарий:** Создаёт опрос в граф-конструкторе → вручную рассылает персональные ссылки клиентам → отслеживает прохождение → анализирует NPS, распределение ответов, heatmap путей, satisfaction matrix
- **«Aha!»-момент:** Первый собранный опрос с инсайтами из heatmap и ответов — видишь, где продукт проседает

### Secondary Users

**Респонденты — клиенты SaaS-продукта**

- **Контекст:** Получают персональную ссылку на опрос вручную (не по email из системы)
- **Взаимодействие:** Welcome-экран с выбором языка → пошаговое прохождение (один вопрос за раз) → финальный экран «Спасибо»
- **Потребности:** Быстро и удобно пройти опрос, mobile-friendly, навигация только вперёд
- **Не являются пользователями системы** — только проходят опросы по ссылке

### User Journey

1. **Создание опроса** — визуальный конструктор на React Flow, настройка вопросов и бранчинга
2. **Добавление респондентов** — ручной ввод email через веб-интерфейс, генерация персональных ссылок
3. **Распространение** — ручная рассылка ссылок клиентам (вне системы)
4. **Мониторинг** — отслеживание статусов прохождения и drop-off
5. **Анализ** — NPS дашборд, распределение ответов, heatmap на графе, satisfaction matrix
6. **Инсайты** — выявление проблемных зон продукта на основе данных

## Success Metrics

- **Получение результатов фидбэков** — продукт работает, когда ты регулярно получаешь данные об удовлетворённости клиентов твоего SaaS
- **Completion rate респондентов** — процент респондентов, дошедших до конца опроса, как индикатор качества UX прохождения
- **Полный цикл работает** — возможность создать гибкий опрос с бранчингом → запустить → собрать ответы → проанализировать результаты

### Business Objectives

- **Фаза 1 (MVP):** Один пользователь, без ролей и организаций. Запустить первый опрос как можно скорее
- **Фаза 2+ (масштабирование):** Ролевая модель (Admin, Organization Owner, Organization Member), multi-tenant, выход на рынок стартапов
- **Стратегическая цель:** Валидировать инструмент на себе, затем масштабировать если зайдёт

### Key Performance Indicators

- **Опросы созданы и запущены** — хотя бы 1 опрос в первый день
- **Completion rate** — % респондентов, завершивших опрос (целевой показатель: >70%)
- **Ответы собраны** — количество полученных ответов по каждому опросу
- **Инсайты получены** — данные NPS, распределение ответов, heatmap путей доступны для анализа

## MVP Scope

### Core Features

**Конструктор опросов (React Flow):**
- Граф-ориентированный конструктор с нодами и рёбрами
- 5 типов вопросов: NPS, открытый, закрытый, матричный, мульти-селект
- Типизированные выходы на нодах (NPS → detractor/passive/promoter, закрытый → выход на каждый вариант)
- Боковая панель настроек при клике на ноду
- Валидация флоу — обнаружение тупиков и циклов
- Все вопросы обязательны
- Flow хранится как JSONB в PostgreSQL

**Жизненный цикл опроса:**
- Статусы: Draft → Active → Completed → Archived
- Ручная активация кнопкой
- Дедупликация по email/ID
- Переактивация завершённого запрещена
- Удаление запрещено (только архивация)
- Каждый ответ отправляется на backend отдельно

**UX респондента:**
- Welcome-экран → пошаговый формат (один вопрос за раз)
- Прогресс-бар
- Навигация только вперёд
- Mobile-friendly адаптивный дизайн
- Сохранение прогресса в localStorage
- Финальный экран «Спасибо»

**Распространение:**
- Ручной ввод респондентов (email) через веб-интерфейс
- Генерация персональной уникальной ссылки
- Статусы прохождения: не открыл / открыл / в процессе / завершил
- Drop-off трекинг

**Аналитика:**
- Общий дашборд: количество ответов, completion rate, средний NPS
- Распределение ответов по каждому вопросу (гистограмма, pie chart, satisfaction matrix)
- Heatmap путей на React Flow графе
- Фильтры: по дате, по респонденту, по ветке флоу

**UI и навигация:**
- Пагинация на всех списках
- Поиск по опросам (по названию) и респондентам (по email)
- Светлая тема
- Онбординг: пустой дашборд с кнопкой «Создать опрос»

**Аутентификация:**
- Один пользователь, email/password, JWT
- Без ролей, без организаций, без инвайтов

**Технический стек:**
- Frontend: Next.js + React Flow + shadcn/ui + Tailwind + React Query
- Backend: NestJS + TypeORM + PostgreSQL
- Monorepo: Turborepo (apps/web, apps/api, packages/ui, packages/shared)
- API: REST, internal only, Swagger/OpenAPI
- CI/CD: GitHub Actions

### Out of Scope for MVP

- Ролевая модель (Admin, Organization Owner, Organization Member)
- Мультиязычность (EN/UK/IT)
- Версионирование опросов
- Email-рассылка из системы
- In-app виджет, QR, SMS, IVR
- AI-анализ (sentiment, автокатегоризация)
- Брендирование опросов
- Превью опроса перед активацией
- Экспорт данных (CSV, PDF)
- Дублирование опросов
- Вебхуки, rate limiting
- Автотесты, логирование, мониторинг
- GDPR, шифрование данных
- Оффлайн/PWA
- Публичный API
- Уведомления (алерты)

### MVP Success Criteria

- Полный цикл работает: создать опрос → добавить респондентов → запустить → собрать ответы → проанализировать
- Completion rate респондентов > 70%
- Аналитика отображает NPS, распределение ответов, heatmap, satisfaction matrix
- Валидация флоу обнаруживает тупики и циклы

### Future Vision

- **Фаза 2:** Ролевая модель и multi-tenant, мультиязычность, email-рассылка, экспорт данных, превью опроса
- **Фаза 3:** AI-анализ, брендирование, вебхуки, публичный API, дополнительные каналы распространения
- **Долгосрочно:** Платформа для CEM с расширенной аналитикой, интеграциями и масштабированием на рынок стартапов
