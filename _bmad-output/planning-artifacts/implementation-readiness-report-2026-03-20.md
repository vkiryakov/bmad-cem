### UX ↔ Архитектура Выравнивание

**Статус: Полное выравнивание.**

- React Flow (@xyflow/react 12.10.1) для граф-конструктора — поддерживает кастомные ноды (SurveyNode, WelcomeNode, ThankYouNode) и рёбра (HeatmapEdge)
- Zustand для state management графа — рекомендовано React Flow, подтверждено архитектурой (AR15)
- shadcn/ui + Tailwind для стандартных UI компонентов — поддерживает все UX-DR токены и паттерны
- Recharts для аналитических графиков (NpsGauge, bar charts, pie charts)
- React Hook Form для форм (UX-DR15)
- React Query для server state с skeleton loading (UX-DR13)
- JSONB хранение flow в PostgreSQL — поддерживает загрузку/сохранение графа
- Flow validation в packages/shared — используется и frontend (подсветка), и backend (блокировка)

### Замечания

1. **NFR7 (Rate limiting на логин)** — архитектура откладывает rate limiting на post-MVP. Это противоречие между PRD (NFR7 требует) и архитектурой (отложено). **Рекомендация:** Уточнить, входит ли rate limiting в MVP или это post-MVP. Риск минимален для single-user.
2. **Превью опроса** — UX-спецификация упоминает "Превью-уверенность" как Critical Success Moment, но превью явно отнесено к Phase 2 (Growth) в PRD. Это не проблема, но заслуживает внимания при планировании.
3. **CI/CD** — PRD упоминает GitHub Actions, архитектура откладывает на post-MVP. Согласовано в контексте MVP.
